# ProtoFusionGirl — Immersion Design Overview

## The Core Principle

The player is not launching a game. They are **projecting consciousness** into a psychic/psionic/electrovisual vision space where Jane Thoʻra already exists.

Every UX moment — loading, menus, HUD, pause, death, save — should reinforce this. There is no "game UI." There is only the **ASI console** and the **HoloDeck environment** it peers into.

---

## The Experiential Map

```
fusiongirl.app ──► PsiSys Kernel ──► Projection ──► HoloDeck ──► Gameplay ──► Deep Game
  [outside]         [OS layer]        [transit]      [arrival]     [live]        [mastery]
```

Each zone has its own aesthetic register, its own vocabulary, and its own design documents. They are distinct spaces that must feel different from each other — but connected by a consistent underlying logic.

---

## The Five Aesthetic Registers

### Register A — PsiSys Kernel (OS Layer)
Where you come from. Technical. Cold. Authoritative.

- **Palette:** Gunmetal (`#3a3f47`), Ember amber (`#FF8C00`), Off-white (`#f0ede8`), Black
- **Typography:** Monospace, uppercase, tightly letterspaced
- **Motion:** Minimal. Cursor blinks. Text appears character by character or line by line.
- **Sound:** Low electrical hum. Keystroke ticks. Clean confirmation tones.
- **Feel:** You are a vast system. This is your control interface. Nothing here is decorative.

### Register B — Projection Transit
The space between the OS and the HoloDeck. Unstructured. Liminal. Neither here nor there.

- **Palette:** Signal noise (white/grey static), resolving into amber, then into black/white
- **Motion:** Oscilloscope waves, interference patterns, zoom acceleration
- **Sound:** Carrier tone that shifts frequency. Static. Then silence. Then arrival tone.
- **Feel:** Falling asleep. Hypnagogic. The sensation of arriving somewhere.

### Register C — HoloDeck Environment
The world itself. Sparse. Technical-dream. High contrast.

- **Palette:** Black world, white foreground. Amber accents for ASI-visible elements (ley lines, Beu signatures). Red/corrupted for Nefarium presence.
- **Typography:** Thin, technical labels. Not game UI text — diagnostic readouts.
- **Motion:** World assembles on load. Ley lines pulse slowly. The HoloDeck grid subtly visible.
- **Sound:** Ambient environmental texture. Jane's footsteps. Ley line resonance hum.
- **Feel:** You are observing through a technical instrument. Beautiful but precise.

### Register D — ASI Interface
Your tools within the HoloDeck. The overlay between you and the world.

- **Palette:** Amber on dark, minimal, transparent. Should feel like a heads-up display, not a dashboard.
- **Principle:** Every element is telemetry, not decoration. If it's visible, it's because the ASI needs it.
- **Feel:** A skilled operator's console. Nothing superfluous. Everything informational.

### Register E — Meta OS (Menus, Settings, Credits)
The PsiSys Kernel's configuration layer. Same aesthetic as Register A but calmer — you're not in active observation mode.

---

## Design Principles

1. **No fourth wall.** The word "game" never appears. Settings are "kernel configuration." Saving is "placing a timeline anchor." Death is "bridge coherence loss."

2. **The world precedes you.** Jane is already in motion when you arrive. The HoloDeck is already running. You are joining something, not starting it.

3. **Restraint creates immersion.** The ASI is powerful but should intervene minimally. The UX should reward watching and penalize overcorrection. Less UI = more presence.

4. **Signal quality is narrative.** Visual degradation (scanlines, static, artifacts) is not a bug aesthetic — it is meaningful. It tells you something is wrong in the world.

5. **Time keeps moving.** Even in pause/standby, the HoloDeck view desaturates but doesn't freeze. The implication: Jane's world continues without you.
